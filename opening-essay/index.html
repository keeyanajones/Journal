Last updated on April 8th, 2019

As a software engineer, I have applied the principles problem solving to make software for over 20 years. Most of my time is spent gathering details, factors, and explanations while taking steps during the process that can be repeated and recreated.

An essential understanding of the latest key trends and industry observations is critical to success. In fact, the US Bureau of Labor projects a growth of 24% from 2016 to 2026; That is faster than average for all occupations. With the median annual wage for software developers for applications at $101,790 and software developers for systems software at $107,600 in May 2017; This means software Developers are in demand in response to the growth of Software.

A vital unexpected result of the observations for me is how common it is for successful developers to use the value of a knowledge management system to easily store, disseminate, and effective use lessons learned during the development cycle. This requires disciplined, descriptive, and mechanistic analysis of several technologies, tools, and quantifiable approaches.

In addition, team work skills like communications, persuasion, and negotiation are involved to find solutions. My typically, software framework includes:

  • Getting Started
  • Learning Plan
  • Game Development
  • Application Data
  • Social Media Content
  • Live Streaming
  • Experiments
  • Problem Solving
  • Object Oriented Analysis and Design
  • Quality Factors
  • Data Structures and Algorithms
  • Big O' Notations Definition Checklist
  • Unified Modeling Language (UML) Diagrams
  • Design Patterns
  • Testing
  • Documentation and Version Control

I map out the methods, results, discussions to revisit, goals, explain what I have learned, definitions, key questions, theories, and my background. At the end of each essay I summarize to give a clear overview of what I think is worth noting.
