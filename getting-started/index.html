Getting Started

Last updated on October 22nd, 2019

  1. Home
  2. Getting Started

At first glance, Software Engineering and Game Development looks like fun. You get to be on the computer regularly, using internet playing games, and using social media.

The truth is the typical day to day activities are all about solving problems with technology, designing efficient systems to meet needs and working with others in some way. Typically the skills needed include designing and developing software applications, coding, debugging, testing and troubleshooting throughout the application development process.

Often a bachelor's degree in Computer Science is required, however, I have found that self taught skills in technology, previous experience building software, and a passion for learning is more important. The ideal Engineer is always learning and eager to work with others. As I became more experienced, I see the value in a learning plan to help.

Below I quickly outline the topics and references in the Appendix that make up this Github version of my coding journal.

A Life Long Learning Plan

Learning is hard, time consuming, and usually does not come naturally. Taking the time to stop and think about what we are learning and how to adapt it is very important. This allows us to build confidence and course correct.

The Engineering Process

There is no definitive “one size fits all” process for the software engineering process and in gaming its even more complex. There are a number of popular approaches based on traditional methods however, when these are applied they often require constant adjustments based on timing, scale, technology, budget and other business factors.

Normally, a combination of traditional approaches in distinct incremental phases is applied simultaneously or iterations until deployment. These phases are usually referred to as the software development life cycle (SDLC). In practice, the approach is selected by the management team in the planning phase of the project.

Thus, an understanding of various process models, techniques, and technologies is essential to the development team. This allows for a cohesive development environment when choosing an integrated development environments, design patterns, paradigms, rules, frameworks, libraries, and models.

Often the steps in getting started include:

  • Developing a plan
  • Exploring possibilities
  • Conducting analysis
  • Engaging and evaluating
  • Communicating and listening
  • Creating a working thing
  • Developing a system
  • Summarizing
  • Avoiding issues
  • Checklist and activities

Next, apply problem Solving to:

  • Probe and Read the problem completely twice
  • Define the problem completely twice
  • Determine the problems seriousness
  • Analyze causes, explore context, and think creatively
  • Brainstorm possible solutions, Generate, and evaluate
  • Choose the best solution alternatives and map out support
  • Solve the problem manually with 3 sets of sample data implementing the solution
  • Optimize the manual steps
  • Write the manual steps as comments or pseudo-code
  • Replace the comments or pseudo-code with real code
  • Optimize the code
